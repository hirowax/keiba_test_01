# ファクター再現性監査（2026-06-11）

**検証スクリプト**: `analyze_factor_audit.py`
**データ**: 70日分（2025-10-04〜2026-06-07）/ 3指数重複馬 1,350頭
**手法**: docs/anaba_research_log.md の教訓に準拠
- 時系列3分割 + 直近期間で全期間プラスのリフトのみ「◎再現」と判定
- 単勝ROIは外れ値（勝ちオッズ上位2頭）除去後を併記
- 複勝ROI（place_oddsあり46日・8頭以上レース・分母は全対象馬）
- 人気帯コントロール（1-3 / 4-6 / 7-18人気の帯内リフト）で人気の代理変数を排除

## ⚠️ 最重要発見: 過去データの前走系ファクター汚染

**2026-03-28 より前の48日分の pickup_scores.json は前走系ファクターが汚染されている。**

- `scrape_horse_prev()`（scrape_prev_data.py / run_pickup_all.py）は馬ページの**最新行**を取るだけで、対象日を考慮しない
- 2026年3月の `run_history_batch.py` 遡及生成、および全日付 `rescore.py` 実行により、
  過去日付の「前走」が**対象レースより未来のレース**で計算された
- 症状: 2026-03-28 より前の全日付で 中4週以内・データ分析ピックアップ・出走馬分析 が N=0。
  前走指数・前走好走・同距離は「間違った前走」で付与されている
- 検証: クリーン窓（2026-03-28以降）の中4週ラベルは自前履歴と全件整合（不一致26件は全て地方交流等の自前DB取りこぼし側）

### 影響
1. **calibrate_threshold.py は汚染データ込みで閾値を計算している**（run.sh で毎回自動実行）
2. analyze_roi.py の全期間集計（中4週 回収283.7% 等）は信頼できない
3. 過去の +2 昇格判断（中4週・前走指数レース内1位）の根拠データに汚染が含まれる

### 修正案（未実施・要承認）
1. `scrape_horse_prev()` を date-aware に修正（対象日より前の最初の行を選ぶ）
2. `rescore.py` はグローバル horse_db ではなく per-date `prev_data.json` を使う
   （現状 prev_data.json は 20260328 のみ存在 → run_pickup_all.py で毎回スナップショット保存する）
3. `calibrate_threshold.py` / `analyze_roi.py` をクリーン日付（≥2026-03-28）に限定

### その他のデータ品質問題
- `last3f` が全72日中71日で未取得（scrape_results.py のバグ）→ 上がり3F系ファクターの研究が不可能
- `corners` が直近の土曜6日分（4/18, 4/25, 5/2, 5/23, 5/30, 6/6）で空 → horse_style.json（逃げ馬判定）が劣化中
- `place_odds` が 2026-01-10〜2026-04-12 の25日分欠損

## 現行ファクターの判定（クリーン窓22日・528頭 + 再構築検証）

再構築 = 自前 race_results 履歴から「対象日より前の直近レース」を再計算（全70日で汚染なし検証可能。ただし前走特定率60%、地方交流等は取りこぼし）

| ファクター | 点 | 判定 | 根拠 |
|---|---|---|---|
| 前走好走 | +2 | **◎維持** | 全期間lift+33pp、全人気帯で+7〜+49pp。再構築でも全期間◎ |
| データ分析ピックアップ | +1 | **◎維持** | lift+6〜+14pp全期間、人気帯+9〜+13pp |

| 同距離前走 | +1 | **○維持** | lift+1〜+6pp、再構築でも全期間◎ |

## 新ファクター候補（再現性検証済み）

### 採用推奨

**1. 3指数すべて1位（+1）** — 最有力
- N=225 勝率17.3% 3着内48.9%（ベースライン35.0%）
- lift: P1+15.5 / P2+16.3 / P3+10.8 / 直近30日+6.2pp（全期間プラス）
- 人気帯: 1-3人気+11pp / 4-6人気+13pp（人気の代理変数ではない）
- 3指数順位はトリプルCSV由来でレース固有 → 過去日付も汚染なし、全70日で検証済み

**2. 距離延長>50m（-1 減点）**
- N=150 3着内24.0%（ベースライン34.8%）
- lift: -10.5 / -18.1 / -7.3 / 直近-7.5pp（全期間マイナスで一貫）
- 人気帯: 1-3人気-15pp / 7-18人気-9pp
- 再構築由来 → 汚染なし。同距離+1と合わせ「距離変化の方向」を完全に表現

**3. 前走10着以下（-1 減点）**
- N=126 3着内20.6% lift: -8.7 / -13.4 / -15.9 / 直近-19.7pp（全期間マイナス）
- 人気帯: 全帯マイナス（-13/-2/-9pp）

### 不採用（検証の結果）
- **指数の絶対値**（近走平均≥90/95、当該距離≥105）: 全てliftマイナス。「3指数重複馬の中では相対順位が効き、絶対値は効かない」
- 距離短縮>50m: 3着内liftマイナス、単ROIは外れ値依存で不安定
- 中2週以内・間隔29-56日・休み明け: 不安定またはN不足
- 芝⇄ダ替わり・枠順・道悪・馬場: 不安定またはN不足
- 前走1着単独: N=46で勝率6.5%と意外に弱い（人気先行で過大評価されやすい）。現行の前走好走（人気1-6×1-3着）の方が優秀

### レース条件の知見（スコアではなく表示・運用への示唆）
- **新馬/未勝利クラス: lift+4.4〜+9.7pp全期間、複勝ROI93.3%（全セグメント中最良）**
  → 現行UIの「1-3R参考バッジ（荒れやすい）」はデータと逆。見直し候補
- 2勝クラス以上: lift-3.1〜-7.7ppで一貫してマイナス → むしろ上級条件こそ「参考」扱いが妥当
- 少頭数(≤10頭): lift+2.8〜+9.1pp全期間プラス
